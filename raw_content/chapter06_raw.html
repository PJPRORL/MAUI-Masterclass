                          # Navigatie

                          In dit hoofdstuk leer je wat meer over navigeren binnen .NET Maui. Je krijgt antwoord op de vragen:

                          • Hoe navigeer je met een knop naar een menuitem.
                          • Hoe navigeer je naar een andere pagina met behulp van een knop?

                          # navigatie naar een menuitem

                          Navigatie naar een pagina via een flyout-menu ken je reeds. Wil je echter, met een knop op een pagina, navigeren naar een andere pagina uit het Flyoutmenu dan moet je een route aanmaken in AppShell.xaml. (regels 4 en 8) Deze route zorgt ervoor dat de Shell correct kan navigeren naar de juiste pagina.

                              <ShellContent
        Title="Home"
        ContentTemplate="{DataTemplate views:HomePage}"
        Route="HomePage" />
    <ShellContent
        Title="Labels"
        ContentTemplate="{DataTemplate views:LabelsPage}"
        Route="LabelsPage" />

                          Deze route kun je aanspreken in de code van een [RelayCommand].

                              [RelayCommand]
    private async void GoHome()
    {

    }
                          • Shell.Current: Verwijst naar de huidige instantie van de Shell, een onderdeel van de .NET MAUI Shell-architectuur die een manier biedt om pagina's te organiseren en te navigeren.
                          • GoToAsync: Een methode om asynchrone navigatie uit te voeren.
                          • "../HomePage": De route waarnaar genavigeerd moet worden. Het "../"-gedeelte geeft aan dat er Ã©Ã©n niveau omhoog in de navigatiehiÃ«rarchie wordt bewogen.

                          # navigatie naar een pagina

                          Niet elke pagina heeft een menuitem. Als je wil navigeren naar zo'n pagina, moet je ook een route registreren, maar dat doe je in AppShell.xaml.cs

                          public partial class AppShell : Shell
{
	public AppShell()
	{
		InitializeComponent();

        Routing.RegisterRoute(nameof(HomePage), typeof(homePage));
    }
}

                          • Routing.RegisterRoute(...): Dit is de methode om een route te registreren in het routingmechanisme van een .NET MAUI-applicatie. Een "route" is een URL-patroon dat overeenkomt met een bepaalde pagina in de app.
                          • typeof(HomePage): Het gebruik van typeof is nodig om het specifieke type van de doelpagina aan te geven.
                          • De nameof operator zorgt ervoor dat je altijd de juiste naam van de pagina gebruikt, zelfs als je de naam van de klasse hernoemt. Dit helpt bij het vermijden van fouten door typefouten.

                          Het [RelayCommand] ziet er nu als volgt uit:

                              [RelayCommand]
    private async void GoHome()
    {
        await Shell.Current.GoToAsync(nameof(HomePage));
    }

                          # Argumenten doorgeven aan een andere pagina

                          Op het moment dat je navigeert naar een andere pagina en gegevens wil meegeven als argument, maak je een Dictionary aan waar je alle objecten inplaatst.

                          In het onderstaande voorbeeld maak je een [RelayCommand] voor navigatie naar een DetailsPage en geef je een object Werknemer mee.

                          [RelayCommand]
private async Task GoToDetails()
{
    if (werknemer == null)
        return;
    await Shell.Current.GoToAsync(nameof(DetailsPage), true, new Dictionary<string, object>
        {
            {"Werknemer", werknemer }
        });
}
                          • true: Dit zorgt ervoor dat de "terugnavigatiegeschiedenis" actief is zodat je terug naar de vorige pagina kunt.
                          • new Dictionary<string, object> { { "Werknemer", werknemer } }: definieert een dictionary met gegevens die moeten worden doorgegeven naar de doelpagina. In dit geval wordt een object van het type "Werknemer" doorgegeven met de sleutel "Werknemer", zodat de doelpagina de details van de werknemer kan weergeven.

                          In deze Dictionary kun je ook meerdere argumenten opnemen. Hieronder vind je de algemene schrijfwijze voor een Dictionary:

                              new Dictionary<string, object>{
        {"Titel1", Object1},
        {"Titel2", Object2},
        ...
    }

                          In het DetailsViewModel kun je de argumenten ophalen met het kenmerk [QueryProperty].

                              namespace MauiIntroductie.ViewModels
    {
        [QueryProperty(nameof(Werknemer),"Werknemer")]
        public partial class DetailsViewModel: BaseViewModel
        {
            [ObservableProperty]
            Werknemer werknemer;

        }
    }

                          Let op! De parameters van de [QueryProperty] staan omgedraaid ten opzichte van de Dictionary. De algemene schrijfwijze is:

                              [QueryProperty(nameof(Object1),"Titel1")]
    [QueryProperty(nameof(Object2),"Titel2")]

                          Opmerking! Het inladen van de argumenten gebeurt nÃ  het uitvoeren van de constructor van je ViewModel. Wil je bv nog een controle uitvoeren of een bewerking uitvoeren van je argument, doe je dit in een onPropertyChanged.

                              [QueryProperty(nameof(Werknemer),"Werknemer")]
    public partial class DetailsViewModel: BaseViewModel
    {
        [ObservableProperty]
        Werknemer werknemer;

        partial void OnWerknemerChanged(Werknemer value)
        {
            if (value != null)
            {
                Shell.Current.DisplayAlert("Fout", "De werknemer is niet correct geladen", "Doorgaan");
            }
        }
    }

                          # Navigatie met GestureRecognizers

                          Hierboven staan de standaard navigatiemogelijkheden maar in de termen van MultiPlatform is het interessant om het ook te hebben over GestureRecognizers. Dit is een eigenschap van een control die herkent wat voor beweging of gebaar je uitvoert op de control. Zo kun je tappen, swipen, drag-drop of pinchen. Je voegt een GestureRecognizer toe in de xaml-code van je pagina.

                          Achter deze beweging programmeer je dan een methode met het kenmerk [RelayCommand] in je ViewModel.

                          Dit is een voorbeeld van een TapGestureRecognizer die de DetailsPage oproept als je op een werknemer tapt:

                                  <CollectionView ItemsSource="{Binding Werknemers}" SelectedItem="{Binding Werknemer}" SelectionMode="Single">
            <CollectionView.Header>
                <Grid ColumnDefinitions="*,*,*,*,*">
                    <Label Text="Id" Grid.Column="0" FontSize="Small"/>
                    <Label Text="Naam" Grid.Column="1" FontSize="Small"/>
                    <Label Text="Functie" Grid.Column="2" FontSize="Small"/>
                    <Label Text="Geboortedatum" Grid.Column="3" FontSize="Small"/>
                    <Label Text="In dienst sinds" Grid.Column="4" FontSize="Small"/>
                </Grid>
            </CollectionView.Header>
            <CollectionView.ItemTemplate>
                <DataTemplate x:DataType="model:Werknemer">
                    <Grid ColumnDefinitions="*,*,*,*,*">
                        <Grid.GestureRecognizers>
                            <TapGestureRecognizer
                                Command="{Binding GoToDetailsCommand, Source={RelativeSource AncestorType={x:Type viewmodel:WerknemerViewModel}}}"/>
                        </Grid.GestureRecognizers>
                        <Label Text="{Binding Id}" Grid.Column="0" />
                        <Label Text="{Binding VolledigeNaam}" Grid.Column="1"/>
                        <Label Text="{Binding Functie}" Grid.Column="2"/>
                        <Label Text="{Binding Geboortedatum, StringFormat='{0:dd-MM-yyyy}'}" Grid.Column="3"/>
                        <Label Text="{Binding InDienst, StringFormat='{0:dd-MM-yyyy}'}" Grid.Column="4"/>
                    </Grid>
                </DataTemplate>
            </CollectionView.ItemTemplate>
        </CollectionView>

                          De TapGestureRecognizer is hier toegevoegd aan de grid. Dit wil zeggen dat als je op een item van de grid tapt, de methode GoToDetailsCommand wordt uitgevoerd. Op dit niveau van Binding zit je echter in het Model Werknemer en niet meer in je ViewModel WerknemerViewModel. Om de methode GoToDetails aan te kunnen spreken, moet je aan .NET MAUI vertellen waar die methode staat. Dit doe je met Source={RelativeSource AncestorType={x:Type viewmodel:WerknemerViewModel}}.

                          RelativeSource is een manier om aan te geven dat de bron van de binding relatief is aan de positie van het element in de visuele boom. In dit geval wil je binden aan een eigenschap in een voorouder-ViewModel.

                          AncestorType wordt gebruikt om aan te geven welk type voorouder in de visuele boom moet worden gebruikt als de bron van de binding. Dit is handig wanneer je een ViewModel gebruikt als gegevenscontext voor een bepaald deel van je UI.

                          Met GestureRecognizers kun je bv ook een methode aanspreken om een werknemer te verwijderen als je naar rechts swipet. Of je kunt een item van de ene CollectionView slepen naar een andere CollectionView met drag en drop.
